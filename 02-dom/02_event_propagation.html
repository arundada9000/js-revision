📖 How Event Propagation Works

1. Three Phases

CAPTURE ↓  (outer → inner)
TARGET  🎯 (the clicked element)
BUBBLE  ↑  (inner → outer)

Every DOM event travels through all three phases. Listeners can opt into capture by setting {capture: true}.

2. event.target vs currentTarget

event.target = the element that was actually clicked.
event.currentTarget = the element whose listener is currently running.

3. stopPropagation()

Stops the event from traveling further. Useful when a child's click shouldn't reach parent listeners.

4. preventDefault()

Blocks the browser's built-in behavior (navigation, form submission, etc.) without stopping propagation.

5. Event Delegation

Attach one listener to a parent and handle child events via event.target. Efficient for dynamic lists.

6. once & passive

{once: true} - auto-remove after first invocation.
{passive: true} - improves scroll performance.
