# Contributing to JavaScript Revision

Thank you for your interest! This is a personal learning repo, but contributions that improve the educational value are welcome.

## How to Contribute

1. Fork the repository
2. Create a feature branch (`git checkout -b my-improvement`)
3. Make your changes
4. Test that everything still works
5. Submit a pull request

## Guidelines

- Keep code self-documenting - minimal comments, meaningful names
- Follow the existing file structure and naming conventions
- HTML files should be self-contained (no external dependencies)
- JS files should run with just `node <filename>`
- If you add a new topic, add a corresponding file in the appropriate section
- Update the relevant README or docs if needed

## Code Style

- 2 spaces for indentation
- Semicolons
- Single quotes for strings
- Meaningful variable names

## Reporting Issues

Open an issue with:
- A clear description
- Steps to reproduce (if applicable)
- Expected vs actual behavior
